# 용어집 (Glossary)

이 문서는 SBEE Lab 인프라 관련 문서 전반에서 사용되는 주요 용어들을 알파벳 순서로 정리하고 설명합니다.

---

### A

**Apptainer (Singularity)**
: 고성능 컴퓨팅(HPC) 환경에 최적화된 컨테이너 플랫폼입니다. 이전 이름은 Singularity였습니다. 연구 환경의 재현성을 보장하고, 보안적인 이점을 가지며, 단일 파일 형태(.sif)로 이미지를 쉽게 이식할 수 있어 우리 인프라의 표준 컨테이너 기술로 사용됩니다.

### D

**Declarative (선언적)**
: "어떻게(How)" 할지를 단계별로 명령하는 '명령형(Imperative)' 방식과 달리, "무엇(What)"을 원하는지, 즉 **바라는 최종 상태**를 기술하는 프로그래밍 및 설정 패러다임입니다. NixOS는 시스템의 최종 상태를 코드로 선언하면, 그 상태에 도달하기 위한 과정은 Nix가 알아서 처리합니다.

**disko**
: 디스크 파티셔닝, 포맷, 마운트 등 디스크 레이아웃을 선언적으로 관리하는 NixOS 모듈입니다. 서버를 처음 설치할 때 디스크 구성을 자동화하고 재현 가능하게 만듭니다.

### F

**Flake**
: Nix의 새로운 표준 패키징 방식으로, 프로젝트의 모든 의존성을 `flake.lock` 파일에 고정하여 **완벽한 재현성**을 보장합니다. 외부 의존성(특정 버전의 nixpkgs 등)과 프로젝트의 출력(NixOS 설정, 패키지 등)을 명확하게 정의합니다.

**flake-parts**
: 복잡해질 수 있는 `flake.nix` 파일을 여러 조각(part)으로 나누어 더 구조적이고 읽기 쉽게 작성할 수 있도록 도와주는 도구입니다.

### I

**IaC (Infrastructure as Code)**
: '코드로 관리되는 인프라'라는 의미로, 서버, 네트워크, 스토리지 등 인프라 자원을 수동으로 설정하는 대신, 코드를 통해 정의하고 프로비저닝하는 관행을 말합니다. 본 프로젝트의 핵심 철학입니다.

**Invoke**
: Python으로 작성된 태스크 러너(task runner)입니다. `inv deploy`, `inv install`처럼 길고 복잡한 셸 명령어들을 간단한 명령으로 추상화하여, 반복적인 관리 작업을 자동화하는 데 사용됩니다.

### M

**Module (NixOS Module)**
: 관련된 NixOS 설정들을 하나의 재사용 가능한 단위로 묶은 것입니다. 모듈은 사용자에게 간단한 옵션(option)을 제공하고, 그 옵션 값에 따라 복잡한 내부 설정을 동적으로 구성합니다. `users`, `sshd` 등이 모두 모듈의 예시입니다.

### N

**Nix**
: 순수 함수형 패키지 관리자입니다. 의존성 지옥(dependency hell) 문제를 해결하고, 원자적(atomic) 업그레이드 및 롤백을 지원하며, 동일한 코드는 항상 동일한 결과물을 만들어내는 것을 보장합니다.

**NixOS**
: Nix 패키지 관리자를 기반으로 만들어진 리눅스 배포판입니다. 운영체제의 모든 설정(사용자, 서비스, 네트워크 등)을 단일 설정 파일(`configuration.nix`)을 통해 선언적으로 관리할 수 있습니다.

### R

**Reproducible (재현 가능한)**
: 동일한 입력(코드)이 주어졌을 때, 언제, 어디서, 누가 실행하든 항상 정확히 동일한 결과물을 만들어내는 특성을 의미합니다. Nix와 Flake가 보장하는 가장 중요한 가치 중 하나입니다.

### S

**sops-nix**
: Mozilla의 `sops`를 NixOS와 통합하여, 암호화된 비밀 파일(secrets)을 선언적으로 관리할 수 있게 해주는 도구입니다. API 키, 비밀번호 등의 민감 정보를 Git 저장소에 안전하게 보관할 수 있게 합니다.

### T

**Terraform**
: 외부 클라우드 서비스나 SaaS 플랫폼의 리소스(DNS 레코드, GitHub 저장소 설정 등)를 코드로 관리하는 IaC 도구입니다. NixOS가 서버 '내부'를 관리한다면, Terraform은 서버 '외부'의 자원을 관리합니다.

**Terragrunt**
: Terraform을 더 편리하게 사용하기 위한 래퍼(wrapper) 도구입니다. 코드 중복을 줄이고, 여러 환경의 원격 상태(remote state)를 쉽게 관리하는 데 도움을 줍니다.

### W

**WireGuard**
: 현대적이고 빠르며, 간결한 VPN 프로토콜입니다. 우리 인프라에서는 물리적으로 분산된 서버들을 하나의 안전한 내부 사설망으로 묶는 데 사용됩니다.
